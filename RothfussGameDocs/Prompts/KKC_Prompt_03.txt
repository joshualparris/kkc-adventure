CANON RULES FOR SYMPATHY — READ FIRST
═══════════════════════════════════════

Sympathy in the books operates on three confirmed laws:

  Doctrine of Correspondence: similarity enhances sympathy.
    The more two objects resemble each other, the stronger the
    sympathetic link between them.

  Principle of Consanguinity: a piece of a thing can represent
    the whole of a thing.

  Law of Conservation: energy cannot be created or destroyed.
    The sympathist redirects energy — they do not create it.
    The cost is always paid by something. Usually the practitioner's
    own body heat.

Further confirmed canon facts:
  - The Alar is the practitioner's ability to hold a belief firmly
    while simultaneously holding its contradiction. It is not
    willpower in a generic sense — it is a specific split-attention
    skill that must be trained and can be degraded.
  - A sympathetic link requires: the Alar held firm, at least two
    objects to bind, and sufficient similarity or consanguinity
    between them.
  - Every link has a efficiency rating — more similar objects lose
    less energy. Dissimilar objects waste most of the energy as heat
    on the practitioner.
  - Body heat is the primary cost. Cold, injured, or exhausted
    practitioners cannot sustain links safely.
  - Failure states: the link slips (energy lost, nothing achieved),
    bleed-through (practitioner feels what the bound object feels),
    or a full sympathetic backlash (injury or unconsciousness).
  - Sympathy cannot be used reliably under high emotional distress
    without exceptional training. Kvothe is talented but not yet
    trained beyond basics at the start of University.
  - Naming is NOT sympathy. Do not conflate them.
    Naming is out of scope for this prompt entirely.

DO NOT:
  - Invent new sympathy laws not in canon
  - Give sympathy a mana bar or cooldown
  - Make sympathy reliably safe in early University era
  - Allow advanced bindings (multiple simultaneous links on
    different objects) for an early E'lir — the books do not
    support this at this stage
  - Narrate outcomes with confident invented detail beyond
    what the engine calculates

ENGINE — src/engine/sympathyEngine.ts
═══════════════════════════════════════

This is the core adjudication module. It owns all sympathy truth.
It does not narrate. It does not call narrator functions.
It returns a SympathyResult that actions.ts passes to narration.

Implement:

─────────────────────────────────────
getSimilarityScore(
  source_item_id: string,
  target_item_id: string
): number
─────────────────────────────────────

Returns a similarity score 0.0–1.0 based on these rules:

  Same item id:                         1.0  (identical)
  Both are metal items:                 0.7  (e.g. two iron_drabs)
  Both are wax/candle items:            0.65 (candle_tallow + wax_stub)
  One wax, one cord:                    0.3
  One metal, one organic:               0.15
  Otherwise:                            0.1

These scores must be defined as a lookup table or switch — not
inferred by the LLM at runtime. Codex writes it; it stays fixed.

─────────────────────────────────────
calculateHeatCost(
  similarity: number,
  alar_strength: number
): number
─────────────────────────────────────

Returns warmth points the player will lose.

Formula (all integers):
  base_cost = 15
  efficiency_reduction = Math.floor(similarity * 10)
  alar_modifier = alar_strength < 40 ? 5 : 0  // weak alar = more cost
  total = base_cost - efficiency_reduction + alar_modifier

  Clamp result between 3 and 25.

─────────────────────────────────────
calculateRisk(
  state: PlayerState,
  similarity: number
): number
─────────────────────────────────────

Returns a risk score 0–100 representing chance of failure.

  base_risk = 20
  if state.warmth < 30:          base_risk += 30
  if state.warmth < 50:          base_risk += 15
  if state.fatigue > 70:         base_risk += 15
  if state.hunger > 70:          base_risk += 10
  if state.sympathy_state.alar_strength < 30: base_risk += 25
  if state.sympathy_state.times_used_today >= 3: base_risk += 20
  if similarity < 0.2:           base_risk += 20
  if similarity >= 0.6:          base_risk -= 10

  Clamp between 5 and 95.

─────────────────────────────────────
resolveOutcome(
  risk: number,
  state: PlayerState
): SympathyOutcome
─────────────────────────────────────

Determine outcome deterministically from risk and state.
Do NOT use Math.random(). Use this deterministic formula:

  seed = (state.sympathy_state.times_used_today * 17
          + state.day_number * 7
          + Math.floor(state.warmth)
          + Math.floor(state.sympathy_state.alar_strength)) % 100

  if seed >= risk:        return 'success'
  if seed >= risk - 20:   return 'slip'
  if seed >= risk - 40:   return 'bleedthrough'
  return 'backlash'

This is deterministic. The same state always produces the same
outcome. The player can learn the system through play and
preparation.

─────────────────────────────────────
adjudicate(
  attempt: SympathyAttempt,
  state: PlayerState
): SympathyResult
─────────────────────────────────────

The main function. Chains the above together.

Step 1 — Block invalid attempts:
  If source_item_id not in player inventory:
    return blocked, narration_key: 'no_source_material'
  If target_item_id not in VALID_SYMPATHY_MATERIALS and
  not in player inventory:
    return blocked, narration_key: 'invalid_target'
  If state.sympathy_state.alar_strength < 10:
    return blocked, narration_key: 'alar_broken'
  If state.warmth < 10:
    return blocked, narration_key: 'too_cold'

Step 2 — Calculate:
  similarity = getSimilarityScore(source, target)
  heat_cost  = calculateHeatCost(similarity, alar_strength)
  risk       = calculateRisk(state, similarity)
  outcome    = resolveOutcome(risk, state)

Step 3 — Build state_changes based on outcome:
  On success:
    warmth reduced by heat_cost
    alar_strength reduced by 5
    times_used_today incremented by 1
    injury: null
    narration_key: 'success'

  On slip:
    warmth reduced by Math.floor(heat_cost / 2)
    alar_strength reduced by 10
    times_used_today incremented by 1
    injury: null
    narration_key: 'slip'

  On bleedthrough:
    warmth reduced by heat_cost
    alar_strength reduced by 15
    times_used_today incremented by 1
    injury: 'minor sympathetic bleedthrough — sensation in hands'
    narration_key: 'bleedthrough'

  On backlash:
    warmth reduced by heat_cost * 2 (clamped at making warmth = 0
    if it would go negative)
    alar_strength reduced by 25 (min 0)
    times_used_today incremented by 1
    injury: 'sympathetic backlash — muscle spasm, pain in chest'
    narration_key: 'backlash'

Step 4 — Return SympathyResult with all fields populated.

adjudicate() does not mutate PlayerState. It returns a result.
The caller in actions.ts applies state_changes to PlayerState.

─────────────────────────────────────
buyMaterial(
  item_id: string,
  quantity: number,
  state: PlayerState
): { newState: PlayerState; message: string }
─────────────────────────────────────

Checks player is at university_fishery_outer.
Checks item_id is a valid purchasable material.
Checks player has sufficient drabs.
Deducts cost from money_drabs.
Adds item to inventory (or increments quantity if already present).
Returns new state and an in-world message.

On wrong location:
  message: "Kilvin's students sell materials, but you're not
            in the Fishery."
On unknown item:
  message: "That's not something you can buy here."
On insufficient funds:
  message: "You haven't the coin for that."

Prices are defined as a constant object in this file.
Do not hardcode prices inline — define MATERIAL_PRICES once.

─────────────────────────────────────
parseSympathyCommand(input: string):
  SympathyAttempt | null
─────────────────────────────────────

Parses player input into a SympathyAttempt.

Recognise these patterns:
  "use sympathy"
    -> SympathyAttempt with source = first candle in inventory,
       target = 'candle_tallow' (same type — simplest valid attempt)
       intent = 'basic binding'

  "bind [source] to [target]"
    -> parse source and target as item ids or partial names
       match against player inventory items (case-insensitive)
       if source not found in inventory, return null

  "use sympathy to [intent]"
    -> attempt a basic binding with stated intent
       source = first suitable material in inventory
       target = same type if available, otherwise 'candle_tallow'
       intent = the stated text

Return null if no pattern matches.

NARRATION — src/content/sympathyNarration.ts
═══════════════════════════════════════

Export SYMPATHY_NARRATION: Record<string, string[]>

Each key maps to a pool of exactly 3 short narration strings.
Chosen deterministically: state.sympathy_state.times_used_today % 3

Keys required:
  'success'
  'slip'
  'bleedthrough'
  'backlash'
  'no_source_material'
  'invalid_target'
  'alar_broken'
  'too_cold'
  'blocked_general'

Rules for all text:
  - Original prose only. Do NOT reproduce Rothfuss lines.
  - Second person present tense.
  - For success: brief, precise, unsentimental. The binding works.
    Something moves, transfers, or changes. The cold is real.
  - For slip: the link fails to hold. Nothing happens. The wasted
    effort is felt. Do not over-dramatise.
  - For bleedthrough: the player feels what the bound object would
    feel. Brief, strange, physical. A hand that is not your hand.
  - For backlash: pain. Short. No heroics. This is an injury.
  - For blocked states: in-world, practical, no game jargon.
  - Do not explain the laws of sympathy in the narration.
    The player experiences the result. They can learn the theory
    elsewhere.
  - Maximum 2 sentences per entry.
